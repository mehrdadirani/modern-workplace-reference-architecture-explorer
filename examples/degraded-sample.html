Auth and live data notes

          

            
Graph scopes: Directory.Read.All, Policy.Read.All, DeviceManagementManagedDevices.Read.All, CloudPC.Read.All.

            
Log Analytics: use read-only KQL via Invoke-AzOperationalInsightsQuery for AVD or operations signals.

            
Fetcher interface: register window.mwReferenceFetchers.graph to supply this page with a validated contract.

            
Default behavior: sample data stays active until a validated JSON upload or registered live fetcher succeeds.

Inline README

          

            How to re-skin and feed real data
            

              
1. Update the CSS variables in the top-level config section if you want client-brandable colors or a logo slot.

              
2. Keep the same JSON contract. The PowerShell generator can emit a live-shaped JSON sidecar and a stamped HTML copy.

              
3. Uploaded JSON is validated structurally before it is rendered; the file is never executed and no HTML from it is injected.

              
4. For public sharing, keep generic tenant names, regions, and device labels. The default samples are already sanitized fixtures.
